## 변경 의도

원 의도(legacy 단계부터): "시장최저가를 입력하면, **로스트아크 거래소 수수료 +
PVE 전리품 경매 분배 시스템**을 고려해 **얼마로 입찰해야 이득**인지 제시".
그러나 legacy → udp-service 이식 시점까지 계산값과 라벨이 어긋나 있었음.

- 라벨이 "(p-1)/p 비율 분배금" 처럼 보이지만, 표시값은 **공교롭게도 손익분기
  최대 입찰가**(`P × 0.95 × (N-1)/N`) 와 동일했음.
- 8/16인 옵션은 출처상 게임 메커니즘 미확인. 16인 단위 경매가 실제로 작동하는지
  확인 자료 없음.

## 무엇이 바뀌었나

### 인원 옵션

- 4/8/**16**인 → **4/8인** (16인 제거).
- 사용자 결정: 4인 던전 + 8인 레이드 둘 다 커버.

### 출력 포맷

```
시장최저가 100,000골드

[4인 파티]
 자가소비 한도 : 71,250 (71.3%)
 재판매 한도   : 67,687 (67.7%)
 비낙찰 분배   : 16,921

[8인 레이드]
 자가소비 한도 : 83,125 (83.1%)
 재판매 한도   : 78,968 (79.0%)
 비낙찰 분배   :  9,871

※ 거래소 수수료 5% 1회 반영. 한도 = 손익분기 최대 입찰가.
```

### 공식

| 항목                 | 공식                             |
| -------------------- | -------------------------------- |
| 자가소비 최대 입찰가 | `floor(P × 0.95 × (N-1)/N)`      |
| 재판매 최대 입찰가   | `floor(P × 0.95² × (N-1)/N)`     |
| 비낙찰자 분배금      | `floor(자가소비한도 × 0.95 / N)` |

- P: 시장최저가
- N: 파티 인원
- 거래소 수수료율: 5% (1회만 반영 — 우편 중복 수수료 가정 없음)

### Description 텍스트

`"경매장 수수료를 고려해 인원당 적정 입찰가"` →
`"PVE 전리품 경매 손익분기 입찰 한도(시장가 기준)"`
